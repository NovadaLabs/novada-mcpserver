      FAQ常见问题
    

    

      

        
          How is this different from calling the Novada REST API directly?
          这和直接调用 Novada REST API 有什么区别?
        
        

          The MCP endpoint exposes Novada tools through the Model Context Protocol so any MCP-compatible agent (Claude, Cursor, Cline, Codex…) can discover and call them with a structured contract — no client SDK, no per-tool wrapper code. Underneath, it dispatches to the same Novada backend.
          MCP 端点通过 Model Context Protocol 暴露 Novada 工具,任何兼容 MCP 的 Agent(Claude、Cursor、Cline、Codex 等)都能以结构化协议自动发现与调用 — 无需客户端 SDK,无需逐工具封装代码。底层仍由 Novada 后端处理。
        

      

      

        
          Do I need a separate key for MCP?
          MCP 需要单独的密钥吗?
        
        

          No. The same Novada API key works for the hosted MCP, the local npx novada-mcp, and the REST API. Quota counters are per token.
          不需要。同一把 Novada API Key 同时适用于托管 MCP、本地 npx novada-mcp 与 REST API。额度按 Token 计算。
        

      

      

        
          Why can't I use browser tools on the hosted endpoint?
          为什么托管端无法使用浏览器工具?
        
        

          Headless browsers (Playwright) don't run inside serverless edge functions. The hosted endpoint returns 501 with an agent_instruction telling the agent to switch to the local MCP via npx novada-mcp — same key, same tool names.
          无头浏览器(Playwright)无法在 serverless 边缘函数中运行。托管端会返回 501 并附带 agent_instruction,引导 Agent 切换到本地 MCP npx novada-mcp — 同一密钥、同一工具名。
        

      

      

        
          Can I send the key as a header instead of a URL query?
          能否用请求头代替 URL 查询参数传递密钥?
        
        

          Yes. Authorization: Bearer sk-eu-novada-YOUR_KEY against https://mcp.novada.com/mcp works identically. Prefer the header when sharing screens or storing client configs in repos.
          可以。对 https://mcp.novada.com/mcp 使用 Authorization: Bearer sk-eu-novada-YOUR_KEY 效果相同。分享屏幕或将客户端配置纳入仓库时,推荐使用请求头方式。
        

      

      

        
          What's the difference between hosted and npx novada-mcp?
          托管版与 npx novada-mcp 有何不同?
        
        

          Hosted = zero install, paste-a-URL, always latest, but no browser tools. Local npx novada-mcp = runs everything including Playwright browser flows, works offline-from-cloud, but you manage updates yourself.
          托管版 = 零安装、粘贴 URL、始终最新,但不支持浏览器工具。本地 npx novada-mcp = 全功能含 Playwright 浏览器流程、可离线使用,但需自行管理更新。
